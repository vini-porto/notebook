Java is a {{c1::hybrid}} language because it is both compiled to bytecode and interpreted by the JVM
The tool {{c1::javac}} compiles Java source code into {{c2::bytecode}}
The {{c1::JVM}} (Java Virtual Machine) executes Java {{c2::bytecode}} on any platform
The JDK includes three core tools: {{c1::java}} (JVM), {{c2::javac}} (compiler), and jshell (REPL)
In Java, every statement must end with a {{c1::semicolon}} (;)
Code blocks in Java are enclosed in {{c1::curly braces}} ({ })
Java is {{c1::case sensitive}}, meaning foo, Foo, and FOO are treated as different identifiers
In Java, you must declare the {{c1::type}} of a variable when you create it — this is called {{c2::static typing}}
Modern Java allows the keyword {{c1::var}} to let the compiler infer the type from the assigned value
A variable declared as var remains {{c1::statically typed}} — its type is fixed at compile time
The four common primitive types in Java are {{c1::int}}, {{c2::double}}, boolean, and char
An {{c1::array}} in Java has a fixed {{c2::length}} and can only hold one type of data
A {{c1::compile-time error}} is caught by the compiler before the program runs
A {{c1::run-time error}} occurs while the program is executing, not before
A {{c1::logic error}} means the program runs without crashing but produces {{c2::incorrect}} results
